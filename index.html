模块化 module

AngularJS 很重要的一个特性就是实现模块化编程，我们可以创建一个模块， 对页面进行功能业务上的划分。

开发单页应用时，要根据不同的业务划分不同的模块，这样就可以把 JS 代码写到多个文件中。

ng 中模块的引入最重要的目的就是为了解决原来全局定义的控制器污染的问题，还有一个目的就是让我们以模块的形式划分架构。

Angular Directive：指令

当网页加载完毕，AnjularJS 自动开始执行

  • AngularJS 指令是扩展的 HTML 属性，带有前缀 ng-。
  • AngularJS 通过内置的指令来为应用添加功能，

  • AngularJS 同时允许你使用内置指令或者自定义指令，从而构建更加超动感的 HTML。

  • ng-xxx 的属性本身并不是标准中定义的属性

  • 很多情况下无法通过语法校验
  • HTML5 允许扩展的属性，以 data- 开头
  • 在 ng 中可以使用 data-ng- 作为前缀来让网页对 HTML5 有效
  • 二者效果相同

常用的指令

  • ng-app
    • 该指令告诉 AngularJS ，该标签内的元素为 AngularJS 应用程序管理的边界
  • ng-init
    • 该指令初始化 AngularJS 应用程序变量
  • ng-model
    • 该指令把文本框的 value 值绑定到变量 name 上
    • 双向绑定
    • 改变了视图中的数据，模型中的数据也会跟着改变
    • 改变了模型中的数据，视图中的数据也会跟着改变
    • 所以我们只需要关注模型中的数据即可
  • ng-click
    • 给元素绑定一个事件
  • ng-repeat
    • 遍历循环数组
    • 插值表达式，就是把应用程序的变量 name 绑定到某个元素的 innerHTML 上
  • ng-bind
    • 类似于表达式，用法基本一致
  • ng-class
  • ng-hide-show-if
  • ng-src

其他常用指令

  • ng-checked : 单选/复选是否选中，只是单向绑定数据
  • ng-disabled : 是否禁用
  • ng-readonly : 是否只读
  • ng-selected : 是否选中，只是单向数据绑定

$scope

  • ng 内置提供了一个 $scope ，即数据模型
  • $scope 是用来视图和数据之间的胶水、粘合剂
  • 视图和控制器之间的数据桥梁
  • 用于在视图和控制器之间传递数据
  • 用来暴露数据模型（数据、行为）
  • 监视模型数据的变化，做出相应的动作 $scope.$watch

控制器 controller

为了避免全局污染，将控制器放到模块中，如果想要在视图中访问定义好的函数里面的 $scope 数据模型对象，
就需要通过 ng-controller 的形式，指定该全局作用域中的函数名称，通过控制器可以拿到 $scope，
这样就可以直接在所有被 ng-controller 包裹中的元素中使用该函数的 $scope，
ng-controller = ‘函数名’，当解析到这里时，ng 会调用该函数
同时初始化生成一个 $scope 仓库

在单页面应用中，一般是根据不同的业务划分不同的模块
根据不同的功能划分不同的控制器（数据模型对象）
控制器就是为了手动获取到对应的数据模型对象
基本是所有的代码都写在 controller 中，操作数据模型对象

  • ng 中的控制器用来对 $scope 进行操作
    • 包括初始化数据和定义事件响应函数等
  • ng 用来解耦业务逻辑层和视图层的关键
  • controller 操作 $scope，View 则展现 $scope 的内容
  • 传统前端程序中大量复杂的 DOM 操作逻辑都被转变成对 $scope 的操作

服务

  • 内置服务
  • 自定义服务

ng 中将 $scope $window $http 等都叫做服务

在 ng 中，假设同一页面有两个模块，两个模块都有一个 input 标签，需要进行增删改查，

又假设没有后台接口，只能进行本地存储到 localStorage ，那么就会造成数据混乱，

此时使用 ng 的服务，调用这个服务，就可以完成增删改查

将增删改查的方法封装到服务中，在多个页面都可以调用这个服务，进行增删改查

同时 ng 不支持使用全局函数，容易引起全局变量命名冲突污染

所以将各方法定义到服务模块中

服务是在控制器中使用，以依赖的形式注入，ng 会自动帮我们创建服务对象

然后在里面通过 this 给构造函数添加静态方法

总结：

服务就是用来获取数据或者说操作数据的，不关心视图，不关心界面交互
可以将服务定义成全局构造函数的形式，但是会污染全局变量
所以 ng 提供了一种自定义服务的机制，可以基于模块去定义服务，然后在控制器里面使用该服务
这样的话就避免了全局污染的问题
给 app 模块定义一个服务
第一个参数就是服务的名称，也就是你原来的构造函数的名字
第二个参数可以直接传递一个函数（类似于原来控制器的写法）
服务本身也可以使用别的服务
第二个参数，就是一个构造函数，可以直接在里面通过 this 给该构造函数添加静态方法
使用服务的时候，肯定是在控制器里面使用服务，使用的时候，ng 会帮我们自动去创建服务对象
然后就可以直接使用里面通过 this 定义的方法了

框架和库

很多前端开发人员还不弄明白什么是库or框架，在此简单的总结一下

  • 框架：比如 Bootstrap
    • 框架一般是封装了很多繁杂的操作，提供了某种规则、约定的使用方式
    • 开发人员只要遵守这些规则来配置即可实现响应功能，以框架为主导
  • 库：比如 jQuery
    • 库一般提供了一些函数、API
    • 由开发者使用库提供的 API 来开发业务，以开发者为主导

什么是AngularJS

  • 一款优秀的前端高级 JS 框架
  • 可以轻松构建 SPA 应用程序
  • 通过指令扩展了 HTML ，通过表法式绑定数据到 HTML
  • 最大程度解放了 DOM 操作（ng 是第一个真正意义上把前端从刀耕火种带到工业时代的流行框架）

关于 AngularJS 的版本

AngularJS 在1.2之后不再支持IE6/7，在1.3之后不再支持IE8，ng2 已经于2016年8月正式发布，但目前市面上用的还不多，一些老的项目还是在用 ng1。

关于 SPA 单页应用

全称 Single Page Applaction，即单页面网页，传统的网站为多页面模型，然后通过 url 的路径地址请求数据。

SPA 单页面则是通过锚点来指定路径。

SPA 的核心点

  • 锚点
  • 监听锚点的事件变化
  • 获取锚点内容
  • 根据不同的锚点内容做不同的内容改变

单页应用的好处

  • 提高了整个应用程序的响应速度、增加了用户体验
  • 重用资源
  • 方便统一控制和代码重用

锚点的使用

  • 网页内部定位
    • 锚点 id 不是标准方式
    • 可以在要定位的元素上方放一个 name 属性，它的值为 a 连接的 href 地址
  • 改变内容
    • 内部定位的以#开头
    • 改变页面内容的最好区分一下，以#/开头
    • hashchange 事件可以在锚点发生改变是触发
    • 但是这样做很麻烦，不如框架来得快

使用 AngularJS 的场景

  • 重型的后台管理系统
    • 有很多增删改查
    • 繁杂的 DOM 操作处理的页面
  • 单页面应用程序
  • 类似于 canvas、html5 游戏、数据可视化应用，使用 ng 就没有意义了

常用的正则表达式

校验数字的表达式

1 数字：^[0-9]$
2 n位的数字：^\d{n}$
3 至少n位的数字：^\d{n,}$
4 m-n位的数字：^\d{m,n}$
5 零和非零开头的数字：^(0|[1-9][0-9])$
6 非零开头的最多带两位小数的数字：^([1-9][0-9])+(.[0-9]{1,2})?$
7 带1-2位小数的正数或负数：^(-)?\d+(.\d{1,2})?$
8 正数、负数、和小数：^(-|+)?\d+(.\d+)?$
9 有两位小数的正实数：^[0-9]+(.[0-9]{2})?$
10 有1~3位小数的正实数：^[0-9]+(.[0-9]{1,3})?$
11 非零的正整数：^[1-9]\d$ 或 ^([1-9][0-9]){1,3}$ 或 ^+?[1-9][0-9]$
12 非零的负整数：^-[1-9][]0-9”$ 或 ^-[1-9]\d$
13 非负整数：^\d+$ 或 ^[1-9]\d|0$
14 非正整数：^-[1-9]\d|0$ 或 ^((-\d+)|(0+))$
15 非负浮点数：^\d+(.\d+)?$ 或 ^[1-9]\d.\d|0.\d[1-9]\d|0?.0+|0$
16 非正浮点数：^((-\d+(.\d+)?)|(0+(.0+)?))$ 或 ^(-([1-9]\d.\d|0.\d[1-9]\d))|0?.0+|0$
17 正浮点数：^[1-9]\d.\d|0.\d[1-9]\d$ 或 ^(([0-9]+.[0-9][1-9][0-9])|([0-9][1-9][0-9].[0-9]+)|([0-9][1-9][0-9]))$
18 负浮点数：^-([1-9]\d.\d|0.\d[1-9]\d)$ 或 ^(-(([0-9]+.[0-9][1-9][0-9])|([0-9][1-9][0-9].[0-9]+)|([0-9][1-9][0-9])))$
19 浮点数：^(-?\d+)(.\d+)?$ 或 ^-?([1-9]\d.\d|0.\d[1-9]\d|0?.0+|0)$

校验字符的表达式

1 汉字：^[\u4e00-\u9fa5]{0,}$
2 英文和数字：^[A-Za-z0-9]+$ 或 ^[A-Za-z0-9]{4,40}$
3 长度为3-20的所有字符：^.{3,20}$
4 由26个英文字母组成的字符串：^[A-Za-z]+$
5 由26个大写英文字母组成的字符串：^[A-Z]+$
6 由26个小写英文字母组成的字符串：^[a-z]+$
7 由数字和26个英文字母组成的字符串：^[A-Za-z0-9]+$
8 由数字、26个英文字母或者下划线组成的字符串：^\w+$ 或 ^\w{3,20}$
9 中文、英文、数字包括下划线：^[\u4E00-\u9FA5A-Za-z0-9_]+$
10 中文、英文、数字但不包括下划线等符号：^[\u4E00-\u9FA5A-Za-z0-9]+$ 或 ^[\u4E00-\u9FA5A-Za-z0-9]{2,20}$
11 可以输入含有^%&’,;=?$\”等字符：[^%&’,;=?$\x22]+
12 禁止输入含有~的字符：[^~\x22]+

特殊需求表达式

1 Email地址：^\w+([-+.]\w+)@\w+([-.]\w+).\w+([-.]\w+)$
2 域名：[a-zA-Z0-9][-a-zA-Z0-9]{0,62}(/.[a-zA-Z0-9][-a-zA-Z0-9]{0,62})+/.?
3 InternetURL：[a-zA-z]+://[^\s] 或 ^http://([\w-]+\.)+[\w-]+(/[\w-./?%&=]*)?$
4 手机号码：^(13[0-9]|14[5|7]|15[0|1|2|3|5|6|7|8|9]|18[0|1|2|3|5|6|7|8|9])\d{8}$
5 电话号码(“XXX-XXXXXXX”、”XXXX-XXXXXXXX”、”XXX-XXXXXXX”、”XXX-XXXXXXXX”、”XXXXXXX”和”XXXXXXXX)：^((\d{3,4}-)|\d{3.4}-)?\d{7,8}$
6 国内电话号码(0511-4405222、021-87888822)：\d{3}-\d{8}|\d{4}-\d{7}
7 身份证号(15位、18位数字)：^\d{15}|\d{18}$
8 短身份证号码(数字、字母x结尾)：^([0-9]){7,18}(x|X)?$ 或 ^\d{8,18}|[0-9x]{8,18}|[0-9X]{8,18}?$
9 帐号是否合法(字母开头，允许5-16字节，允许字母数字下划线)：^[a-zA-Z][a-zA-Z0-9_]{4,15}$
10 密码(以字母开头，长度在6~18之间，只能包含字母、数字和下划线)：^[a-zA-Z]\w{5,17}$
11 强密码(必须包含大小写字母和数字的组合，不能使用特殊字符，长度在8-10之间)：^(?=.\d)(?=.[a-z])(?=.*[A-Z]).{8,10}$
12 日期格式：^\d{4}-\d{1,2}-\d{1,2}
13 一年的12个月(01～09和1～12)：^(0?[1-9]|1[0-2])$
14 一个月的31天(01～09和1～31)：^((0?[1-9])|((1|2)[0-9])|30|31)$
15 钱的输入格式：

- 有四种钱的表示形式我们可以接受:"10000.00" 和 "10,000.00", 和没有 "分" 的 "10000" 和 "10,000"：^[1-9][0-9]*$ 
- 这表示任意一个不以0开头的数字,但是,这也意味着一个字符"0"不通过,所以我们采用下面的形式：^(0|[1-9][0-9]*)$ 
- 一个0或者一个不以0开头的数字.我们还可以允许开头有一个负号：^(0|-?[1-9][0-9]*)$ 
- 这表示一个0或者一个可能为负的开头不为0的数字.让用户以0开头好了.把负号的也去掉,因为钱总不能是负的吧.下面我们要加的是说明可能的小数部分：^[0-9]+(.[0-9]+)?$ 
- 必须说明的是,小数点后面至少应该有1位数,所以"10."是不通过的,但是 "10" 和 "10.2" 是通过的：^[0-9]+(.[0-9]{2})?$ 
- 这样我们规定小数点后面必须有两位,如果你认为太苛刻了,可以这样：^[0-9]+(.[0-9]{1,2})?$ 
- 这样就允许用户只写一位小数.下面我们该考虑数字中的逗号了,我们可以这样：^[0-9]{1,3}(,[0-9]{3})*(.[0-9]{1,2})?$ 
- 1到3个数字,后面跟着任意个 逗号+3个数字,逗号成为可选,而不是必须：^([0-9]+|[0-9]{1,3}(,[0-9]{3})*)(.[0-9]{1,2})?$ 
    + 备注：这就是最终结果了,别忘了"+"可以用"*"替代如果你觉得空字符串也可以接受的话(奇怪,为什么?)最后,别忘了在用函数时去掉去掉那个反斜杠,一般的错误都在这里

HTML5 的 DOM 操作

dom（document object model 文档对象模型）

当我们打开一个 html 页面时，就会产生一个dom对象，操作dom对象，其实就是在操作 html 的内容

打开浏览器，浏览器的内核去解析 html 网页，解析 html 网页的是从上到下解析，
解析 html 的过程当中，会创建一个dom 对象，把 html 里面内容映射到dom 里面来。

在 dom 看来，html 里面的内容都是节点
html 里面的内容可以分为五种类型的节点

  • 文档节点 整个html 可以当作成一个节点
  • 元素节点 页面里面的所有的标点都可以当作成元素节点
  • 属性节点， 页面上面所有的标签对应的属性都可以理解成属性节点
  • 文本节点 页面里面所有的标签对应内容都是文本 节点
  • 注释节点
    我们做开发都是要操作这些节点，对这些节点进行增删改查，以及样式操作。
    要操作这些节点（元素节点，属性节点，文本节点），我必须先得到元素节点。

数组的 forEach 方法

ES5 新增的数组的 forEach 方法
用来遍历数组
比如

等同于

该方法传入的参数为

[].forEach(function(value,index,array){})

另外forEach循环不能退出，匹配到了还会继续循环，有损性能

对比 jQuery 中的 $.each 方法

$.each([],function(index,value,array){})

他们的传参顺序是不同的

类似的还有 jQuery 中的 map 方法

forEach 方法的特点

forEach 循环不能退出，即使匹配到了也会把后续的循环执行完

这里不能用 break 和 return 来打断

注意： 没有办法中止或者跳出 forEach 循环，除了抛出一个异常。如果你需要这样，使用forEach()方法是错误的，你可以用一个简单的循环作为替代。如果您正在测试一个数组里的元素是否符合某条件，且需要返回一个布尔值，那么可使用 Array.every 或 Array.some。
some 只要一个返回 true 则遍历结束
every 所有都要为 true
