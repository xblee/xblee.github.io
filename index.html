AngularJS学习笔记（一）

Awesome

首先给大家推荐一个网站 [https://www.awesomes.cn/]，这是个 JavaScript 的技术栈，上面会定期会定期更新一些比较火的前端技术。

框架和库

很多前端开发人员还不弄明白什么是库or框架，在此简单的总结一下

  • 框架：比如 Bootstrap
    • 框架一般是封装了很多繁杂的操作，提供了某种规则、约定的使用方式
    • 开发人员只要遵守这些规则来配置即可实现响应功能，以框架为主导
  • 库：比如 jQuery
    • 库一般提供了一些函数、API
    • 由开发者使用库提供的 API 来开发业务，以开发者为主导

什么是AngularJS

  • 一款优秀的前端高级 JS 框架
  • 可以轻松构建 SPA 应用程序
  • 通过指令扩展了 HTML ，通过表法式绑定数据到 HTML
  • 最大程度解放了 DOM 操作（ng 是第一个真正意义上把前端从刀耕火种带到工业时代的流行框架）

关于 AngularJS 的版本

AngularJS 在1.2之后不再支持IE6/7，在1.3之后不再支持IE8，ng2 已经于2016年8月正式发布，但目前市面上用的还不多，一些老的项目还是在用 ng1。

关于 SPA 单页应用

全称 Single Page Applaction，即单页面网页，传统的网站为多页面模型，然后通过 url 的路径地址请求数据。

SPA 单页面则是通过锚点来指定路径。

SPA 的核心点

  • 锚点
  • 监听锚点的事件变化
  • 获取锚点内容
  • 根据不同的锚点内容做不同的内容改变

单页应用的好处

  • 提高了整个应用程序的响应速度、增加了用户体验
  • 重用资源
  • 方便统一控制和代码重用

锚点的使用

  • 网页内部定位
    • 锚点 id 不是标准方式
    • 可以在要定位的元素上方放一个 name 属性，它的值为 a 连接的 href 地址
  • 改变内容
    • 内部定位的以#开头
    • 改变页面内容的最好区分一下，以#/开头
    • hashchange 事件可以在锚点发生改变是触发
    • 但是这样做很麻烦，不如框架来得快

使用 AngularJS 的场景

  • 重型的后台管理系统
    • 有很多增删改查
    • 繁杂的 DOM 操作处理的页面
  • 单页面应用程序
  • 类似于 canvas、html5 游戏、数据可视化应用，使用 ng 就没有意义了

需要掌握

  1. 能概述 AngularJS 是什么
  2. 能概述什么是 SPA 应用程序以及掌握锚点在 SPA 应用程序中的使用
  3. 能掌握 bower 包管理工具的基本使用
  4. 能解释一个基本的 ng 应用程序执行流程（见hello-world代码解析）
  5. 能掌握 AngularJS 中的基本的指令和表达式的使用
  6. 能掌握 AngularJS 中基本的模块使用（创建模块，依赖模块，获取模块）
  7. 能使用 AngularJS 完成模拟用户注册功能

版本控制及 SVN

版本控制可以解决的问题

  1. 历史记录
  • 记录每次更改日志
  1. 多人协作
  • 各自编写代码，最后合并整合

使用的软件

  • 记录一款软件添加或更改源代码的过程
  • 回滚到特定阶段，恢复误删除的文件
  • 合并多人协作的文件等

集中式版本控制 - SVN（Apache Subversion）

术语

  • 源代码库（repository）：源代码统一存放的地方
  • 检出（checkout）：当你手上没有源代码的时候，就需要从 responsive checkout 一份
  • 提交（commit）：当你已经修改了代码，就需要 commit 到 repository
  • 更新（update）：当你已经 checkout 了一份源代码，Update 一下就可以和 repository 上的源代码同步，你手上的代码就会有最新的变更

特点

集中式版本管理将所有的数据集中存放在服务器中，这是有便于统一管理的优点。

但是一旦开发者所处的环境不能连接服务器，就无法获取最新源代码，开发也就无法进行。

服务器宕机时也是同样的道理，而且万一服务器故障导致数据丢失， 恐怕开发者就再也见不到最新的源代码了。

使用 VisualSVN 搭建 SVN 服务器

SVN 服务器分为了命令行工具和图形化工具

为了方便，一般采用图形化工具
  • 创建用户或组
  • 创建版本仓库
  • 设定用户权限

使用 TortoiseSVN 作为 SVN 的客户端

  • 检出项目： checkout
  • 增加文件或目录： add （增加） commit （提交）
  • 修改文件或目录： commit （提交）
  • 删除文件或目录： commit （提交）
  • 更新文件或目录： update （更新）
  • 查看版本日志： log （日志）

建议（防止冲突）

  • 一个文件最好同一时间只被一个人修改提交
  • 多跟团队成员沟通
  • 不要随便去修改别人的文件
  • 使用 SVN 解决冲突很繁琐，现在用的人也越来越少了

版本控制的建议

  • 不要频繁的提交版本
    • 一般有比较成熟的功能模块的时候，再去提交
    • 修复了功能性 bug 的时候再去提交
  • 每次 commit 之前都要 update
    • 因为你在编辑这个文件的时候，可能比人已经编辑并提价了某个版本
    • 所以先 update，目的是为了检查一下服务器上有没有最新版，如果有，直接更新
      *更新的过程中如果遇到冲突，不要慌，去手动解决
  • 每次 commit 的时候都要写提交日志
    • 这个提交日志就好比你保存副本的时候加的一个标记
    • 目的是为了日后做版本的回退查找以及查看记录更新状态

命令行

shell

shell 就是一个程序，接收键盘输入的命令，传给操作系统去执行

几乎所有的 Linux 都提供了一个名为 bash 的 shell 程序

终端仿真器

当使用图形用户界面时，我们需要另一个和 shell 交互的叫做终端仿真器的程序。 在 Windows 上，一般使用操作系统自带的 cmd 或者 powershell。 在 Linux 上，如果是图形用户界面，那么可以使用 terminal 或者 konsole、gnome-terminal 之类 的终端仿真器，但基本上，它们都完成同样的事情，让我们能访问 shell， 你可能会因为附加的一些花俏功能而喜欢上某个终端。

关于名字，如果有人提到：控制台、终端、bash、shell、terminal 等，一般都是指上面这些。

一般在 Linux 上，都会提供一个 bash 或者 terminal 之类的工具，让你来使用 shell

安装 Git for Windows 客户端体验 shell

Git for Windows 客户端自带了一个终端仿真器：git bash

[下载地址：]（https://git-for-windows.github.io/）

git for Windows - github 托管地址 ，然后找到 Realease 模块，找到下载的对应版本即可。

注意：虽然在 Windows 上可以通过 cmd 或 powershell 体验终端的使用， 但是学习终端最正确的方式是学习 Linux 上的命令行，要更纯正，更强大。

所以，我们这里通过使用 Git for Windows 客户端（自带了一个 bash）可以供我们学习 shell。

使用终端操作文件系统

  • 绝对路径和相对路径

    • 移动文件或目录，也可以重命名
  • mkdir （make directory）

  • rm （remove）
    • 删除

    • 删除指定空目录
  • clear 清屏
  • touch 文件名
    • 创建该文件
  • cat 文件名
    • 查看指定的文本文件

文本编辑器 vi

vi 全称 visual interface

类似于 windows 中的记事本 notepad

但比 notepad 强大很多

vi 操作

  • vi 启动vi
  • :q 退出vi
  • :q! 强制退出
  • esc 按两次可以退回普通模式
  • vi 文件名 创建新文件
  • vi 文件名 编辑已有文件
  • i 进入插入模式 insert 后才能编辑文本内容
    • 输入完成后按 ESC 退出插入模式到命令模式

保存工作

  • :w 保存
  • :wq 保存并退出
  • ZZ 保存并退出
  • :w 文件名 指定可选文件名 类似于另存为

基本编辑

插入

  • a 越过行尾，进入插入模式
  • A 直接移动到当前行末尾，进入插入模式
  • o 在当前行的下方打开一行，进入插入模式
  • O 在当前行的上方打开一行，进入插入模式

撤销

  • u 撤销最后一次修改
  • ctrl + r 回复撒谎那个一步被撤销的操作

删除

  • x 删除当前字符

  • d$ 从光标位置开始到当前行的行尾。
  • d0 从光标位置开始到当前行的行首。
  • dˆ 从光标位置开始到文本行的第一个非空字符。
  • dG 从当前行到文件的末尾。

复制

  • yy 当前行。
  • 5yy 当前行及随后的四行文本。
  • yW 从当前光标位置到下一个单词的开头。
  • y$ 从当前光标位置到当前行的末尾。
  • y0 从当前光标位置到行首。
  • yˆ 从当前光标位置到文本行的第一个非空字符。
  • yG 从当前行到文件末尾。
  • y20G 从当前行到文件的第 20 行。
  • J 把行与行之间连接起来

粘贴

  • p 将剪切板中的文本粘贴到光标位置之后
  • P 将文本粘贴到光标之前

查找替换

  • f 要查找的字符
    • 通过输入分号查找当前行下一个
  • / 要查找的字符
    • 输入 n 查找下一个
